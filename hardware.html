Contains hardware specific implementation of device functionality through a 1.3" TFT Bonnet screen for a Raspberry Pi. 

This screen is 240x240 pixels, and the header board also has a joystick(with click-in button) and two other buttons. This contains specific setup of hardware, including accessing the board, setting up the screen, and creating a system for using and rendering interfaces. This functionality was split into four classes:

  • Display - Handles image rendering of all UIs
  • Input - Handles input from the joysticks and buttons
  • Screen - Handles physically setting up screen and buttons, and rendering images
  • UI(Manager) - Manages current state, changing how inputs work and what is rendered
