Hardware

  • Raspberry Pi 5 - 8 GB
  • SD Card - 128 GB
  • Raspberry Pi High Quality HQ Camera (12MP)
  • 16mm 10MP Telephoto Lens for Raspberry Pi HQ Camera (10MP)
  • Adafruit 1.3" Color TFT Bonnet for Raspberry Pi - 240x240 TFT + Joystick Add-on
  • 3-D printed housing and mount
  • Portable, Type-C Power Source (5V)

Assumptions

  • The system has a charged, stable power supply.
  • The system's components are properly plugged in and installed into the housing.

Software

  • Bookworm OS
  • Astrometry.net Local Installation
    • Install index files to directory
  • Python Modules
    • Starplot and Skyfield are key Python libraries for the project, as they allow us to manage targets and render OpticPlots, which emulate what a telescope would see. This will be used on the device to display what should be currently visible through the eyepiece.
    • Picamera2 is used to dynamically control capture settings and take images.
    • Pillow is used throughout the project for general image manipulation and display.

Assumptions

  • Camera can be controlled precisely, including capturing images and setting parameters such as exposure and gain.
  • Solver is properly setup and configured to camera (pixel scale, downsample rate, etc.)
  • Astrometry.net index files are correct and provide accurate solve information.
  • Optic Plots generated by Starplot are accurate and closely represent what would be seen through the scope.

 

Environment

  • User must have a working telescope with a sliding dovetail rail, common for most finderscopes, to mount device.

Assumptions

  • The system will be attached to a telescope.
  • The system will operate under clear skies with minimal cloud cover.
  • The system's camera will be minimally blocked and have a good view of the sky.
